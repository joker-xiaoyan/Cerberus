客户端生成

在服务器上执行 cargo build --release，完成后可在此下载生成的客户端程序。

客户端配置

配置客户端连接的 C2 Host/IP、端口和协议；留空则使用默认值。

启用域前置后，HTTP/HTTPS 请求的 Host 头将使用此域名，而实际 TCP 连接仍使用上方的 IP。
💡 需要配合 CDN（如 CloudFront、Azure CDN）使用，确保 CDN 将请求转发到你的 C2 服务器。

Windows 目标支持所有输出格式和安全特性。 ⚠️ Linux 目标：仅支持 ELF 可执行文件输出，Windows 专属安全特性（Ekko Sleep、ETW/AMSI 规避等）将自动跳过。
💡 Linux 编译通过 Docker 容器交叉编译（需要安装 Docker Desktop），目标为 musl 静态链接（无外部依赖）。

EXE：直接运行的独立程序 | DLL：可反射注入的 DLL (含 Execute 导出) | Shellcode：通过 sRDI 转换的位置无关代码
Stager：极小的下载器，从 C2 拉取完整 Beacon（支持 EXE / DLL / Shellcode 三种格式）

Stager 从 C2 下载完整 Stage 的 URL 路径，应伪装为合法资源（如 .js/.css/.png）。

安全特性

Patch 模式 — 安全特性由模板预编译决定

当前使用二进制补丁模式生成 Beacon，安全特性（Stack Spoof、Indirect Syscall、ETW/AMSI 绕过、ThreadPool 栈伪造、反沙箱、堆加密等） 在模板编译时已固定，无法通过前端选项修改。以下仅显示可在补丁模式下调整的运行时选项。

⚠️ 仅用于开发调试！ 勾选后生成的 EXE 会弹出控制台窗口并输出运行日志。正式部署时请务必关闭此选项。

使用 {} 作为占位符，例如 /static/js/app.{}.js
