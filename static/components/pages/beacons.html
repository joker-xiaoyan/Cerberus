SO插件管理 🐧 Linux

SO 插件开发指南

SO Loader 是 Linux 平台的类 BOF 插件机制，支持内存加载 .so 共享库并执行指定函数。

编译命令：gcc -shared -fPIC -o plugin.so plugin.c

入口函数签名：int go(unsigned char *args, int args_len)

默认入口点依次尝试：go → run → execute → plugin_main

提示：SO 插件在 Beacon 端同进程直接执行（无 fork），通过信号处理器实现崩溃保护。stdout/stderr 输出会被捕获并回传。仅支持 Linux x86_64 Beacon。

插件模板 

// plugin.c
#include <stdio.h>

int go(unsigned char *args, int args_len) {
    printf("[+] Hello from SO plugin!\n");
    if (args && args_len > 0) {
        printf("[*] Args: %.*s\n", args_len, args);
    }
    return 0;
}

编译：gcc -shared -fPIC -o plugin.so plugin.c

                            注入配置
                        

                        

                            

                                
                                
                            

                            

                                
                                
                            

                            

                                
                                
                            

                            

                                
                                
                                

                                    Module Stomping 加载合法签名 DLL 后覆写 .text 段，内存类型 MEM_IMAGE + RX，完全合法外观
                                

                            

                            
                        

                    


                    
                    

                        

                            注入技术说明
                        

                        

                            
                            

                                

                                    
                                    4 — Module Stomping（推荐）
                                    OPSEC 最佳
                                

                                
CreateProcess(SUSPENDED+PPID Spoof) → 加载合法签名 DLL → 覆写 .text 段 → Resume

                                
内存类型 MEM_IMAGE（非 MEM_PRIVATE），保护属性 RX，完全合法外观。避免 NtCreateSection(RWX) 和 VirtualAllocEx 等高危 IOC

                            

                            

                                

                                    
                                    2 — Early Bird APC（推荐）
                                    iOA 规避
                                

                                
CreateProcess(SUSPENDED+PPID Spoof) → Resume → [长延迟] → Suspend → QueueApcThread → Resume

                                
Init-First Pattern 打破 "SUSPENDED→立即注入" 的 EDR 行为模式匹配，含 Remote ETW Patching

                            


                            

                            
全部技术列表


                            

                                
0 — Classic Remote Thread

                                
VirtualAllocEx → WriteProcessMemory → CreateRemoteThread (spoofed start address)

                            

                            

                                
1 — APC Injection

                                
打开已存在线程 → QueueApcThread → 等待线程进入 alertable 状态

                            

                            

                                
2 — Early Bird APC

                                
CreateProcess(SUSPENDED+PPID Spoof) → Init-First Pattern → QueueApcThread → Resume

                            

                            

                                
3 — Section Mapping

                                
NtCreateSection → NtMapViewOfSection(local+remote) → CreateRemoteThread (spoofed)

                            

                            

                                
4 — Module Stomping（进程镂空 + DLL 覆写）

                                
CreateProcess(SUSPENDED+PPID Spoof) → LoadLibrary(签名DLL) → 覆写 .text → SetContext → Resume

                            

                            

                                
5 — Thread Hijacking

                                
SuspendThread → GetContext → SetContext(Rip=shellcode) → ResumeThread

                            

                            

                                
6 — Callback: TimerQueue

                                
VirtualAllocEx → WriteProcessMemory → CreateTimerQueueTimer 回调执行

                            

                            

                                
7 — Callback: EnumWindows

                                
VirtualAllocEx → WriteProcessMemory → EnumWindows 回调执行

                            


                            
                            

                            

                                

                                    OPSEC 增强（所有技术通用）
                                

                                

                                    
• PPID Spoofing — 自动伪造父进程为 explorer.exe

                                    
• Thread Start Spoofing — CreateRemoteThread 起始地址指向 ntdll 合法导出

                                    
• RW→RX 分离 — 严格遵循，永不分配 RWX 内存

                                    
• API Hash Resolution — 所有 API 通过 DJB2 hash 解析，不走 IAT

                                    
• Timing Evasion — API 调用间 300-800ms 随机延迟

                                    
• Remote ETW Patching — patch EtwEventWrite + NtTraceEvent + EtwEventWriteFull
